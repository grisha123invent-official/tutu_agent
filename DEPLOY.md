# Деплой (VPS за границей, доступ из РФ)

## Почему это работает из РФ

Браузер пользователя общается **только с твоим доменом**. В OpenAI (заблокирован в РФ)
ходит твой сервер за границей, а не клиент. MCP Туту — публичный, с зарубежного сервера
доступен. Значит из РФ нужно единственное: чтобы **резолвился и открывался твой домен**.

Поэтому:
- **Без Cloudflare-проксирования.** Оранжевое облако Cloudflare в РФ часто тормозит/блокируется.
  Делаем **A-запись напрямую** на IP сервера. TLS выдаёт Let's Encrypt (через Caddy).
- IP сервера должен быть доступен из РФ (Hetzner, обычно ок). Если провайдер/подсеть попали
  под блок — сменить IP/провайдера.

## Что деплоится

| Компонент | Как |
|---|---|
| Фронт (React) | собирается в статику, раздаётся Caddy |
| Бэкенд (Node WS + OpenAI + MCP) | контейнер `backend`, порт 8787 (только внутри) |
| Caddy | авто-HTTPS + reverse-proxy `/api` и `/ws` на бэкенд |

## Шаги

### 1. DNS
В панели домена создай **A-запись** (proxy выключен, если это Cloudflare — «серое облако»):
```
@   A   <IP_твоего_VPS>
```
(или поддомен: `agent  A  <IP>`). Подожди пропагацию (пара минут).

### 2. На сервере
```bash
# поставь Docker + compose plugin (если ещё нет)
curl -fsSL https://get.docker.com | sh

# забери проект
git clone https://github.com/grisha123invent-official/tutu_agent.git
cd tutu_agent

# домен для Caddy
cp .env.deploy.example .env
nano .env            # DOMAIN=твой-домен.ru

# ключ OpenAI для бэкенда
cp backend/.env.example backend/.env
nano backend/.env    # OPENAI_API_KEY=sk-...

# запуск
docker compose up -d --build
```

### 3. Проверка
```bash
docker compose ps
docker compose logs -f caddy      # увидишь выдачу TLS-сертификата
curl -s https://твой-домен.ru/api/health   # {"ok":true,...}
```
Открой `https://твой-домен.ru` в браузере. Для голоса нужен **https** (микрофон работает
только по защищённому соединению) — Caddy его уже даёт.

### 4. Обновления
```bash
git pull
docker compose up -d --build
```

## Firewall
Открой на VPS порты **80** и **443** (80 нужен Let's Encrypt для проверки домена).
```bash
ufw allow 80 && ufw allow 443 && ufw enable
```

## Если из РФ не открывается
1. Проверь, что домен резолвится в IP сервера и **не** через Cloudflare-прокси.
2. Проверь доступность IP из РФ (ping/curl с российского канала, сервисы типа «check-host.net»).
3. Если IP под блоком — смени IP/провайдера. Опция на будущее: поставить рядом
   российский «фронт»-прокси (nginx на RU-VPS), проксирующий на зарубежный бэкенд, —
   но обычно прямого доступа к нормальному зарубежному IP достаточно.
